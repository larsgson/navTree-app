# navTree-app

A template for building offline-first Android navigation tree apps with semantic search. Built with Astro and Capacitor.

## Quick Start

1. Click **"Use this template"** on GitHub to create your own repo
2. Clone your new repo
3. `cp app.config.example.json app.config.json` and edit with your app's identity
4. Add your content to `src/data/content/` and images to `pictures-src/`
5. `pnpm install && pnpm build`
6. `pnpm run cap:sync && pnpm run cap:open` to build the APK

## Features

- Chapter/section book content with images and tables
- Offline semantic search with two modes:
  - **Mini** — fast, lightweight (Model2Vec, plain JS)
  - **Standard** — higher quality (all-MiniLM-L6-v2, ONNX transformer)
- Client-side full-text search (FlexSearch)
- Mobile-optimized reading experience
- Fully configurable via `app.config.json`

## Configuration

All app-specific settings are in `app.config.json` (gitignored). To get started:

```bash
cp app.config.example.json app.config.json
# Edit app.config.json with your app's values
```

Everything reads from this single file — Astro at build time, the Android project via Gradle, and Capacitor via `capacitor.config.ts`. No manual Android file editing needed.

### Config options

```json
{
  "appId": "com.example.mybook",
  "appName": "My Book Reader",
  "defaultLanguage": "eng",
  "defaultBookFolder": "mybook",
  "branding": { "subtitle": "...", "metaDescription": "..." },
  "search": { "askLabel": "...", "searchingLabel": "...", ... },
  "examples": { "question1": "...", "question2": "...", "question3": "..." }
}
```

## Prerequisites

- Node.js 18+
- pnpm
- `cwebp` (`brew install webp`) — for image conversion
- Android Studio (for building the APK)

## Content Setup

The app expects content and images in these directories (both gitignored):

```
src/data/content/{lang}/{bookfolder}/   # Book content (JSON)
pictures-src/{lang}/{bookfolder}/       # Source images (PNG)
public/assets/embeddings-mini.json      # Embeddings for Mini mode (256d)
public/assets/embeddings-standard.json  # Embeddings for Standard mode (384d)
```

## Development

```bash
pnpm install
pnpm dev
```

## Build

```bash
pnpm build
```

This will:
1. Convert source PNGs to WebP (`scripts/convert-pictures.sh`)
2. Download ML models if not present (`scripts/download-models.sh`)
3. Generate the image index
4. Build the Astro site to `dist/`

## Android APK

```bash
pnpm run cap:sync   # Build + copy to Android project
pnpm run cap:open   # Open in Android Studio
pnpm run cap:run    # Build and run on connected device
```

## Embeddings

The semantic search requires pre-computed embeddings files generated by a backend:
- `public/assets/embeddings-mini.json` (256d, for Mini mode)
- `public/assets/embeddings-standard.json` (384d, for Standard mode)

See `docs/backend-embeddings-export.md` for generation instructions.

## ML Models

Models are downloaded automatically during build:
- `Xenova/all-MiniLM-L6-v2` — 23 MB (Standard mode, ONNX)
- `minishlab/potion-base-8M` — 30 MB (Mini mode, Model2Vec)

To download manually: `bash scripts/download-models.sh`
